## 0. Scope (LOCKED)

### What this slice ships

A new self-contained library package `com.aritr.rova.service.dualrecord` that builds the **pipeline foundation** for true simultaneous dual-stream recording (portrait 9:16 + landscape 16:9) from a single back/front camera, using CameraX 1.4.2's `CameraEffect` API as the camera tap and a custom EGL + dual `MediaCodec` + dual `MediaMuxer` pipeline for the actual encoding. The public surface is a `DualVideoRecorder` class plus supporting types, designed for consumption by `RovaRecordingService` in **Phase 6.1b**.

### What this slice does NOT ship

### Components

1. **`DualVideoRecorder`** (public API) тАФ top-level orchestrator. Holds the `CameraEffect`, manages the encoder/muxer/audio lifecycle, exposes `start(outputs, executor, callback): DualRecording` and `release()`.
2. **`DualSurfaceProcessor`** (internal) тАФ implements `androidx.camera.core.SurfaceProcessor`. Receives the camera's input `SurfaceRequest`, owns the input `SurfaceTexture`. Owns one or more `SurfaceOutput`s (PreviewView's surface, plus our two encoder surfaces). The CameraEffect's contract requires us to provide outputs for the targets we requested; for target=PREVIEW we get PreviewView's expected output, and we additionally render to our owned encoder surfaces.
3. **`EglRouter`** (internal) тАФ the EGL14 context + GLES2 shader program. Builds one shared GL context. Owns the input `SurfaceTexture` consumer side. Renders each incoming camera frame to three targets: (a) PreviewView's expected output, (b) Encoder-P's input surface (with portrait crop transform), (c) Encoder-L's input surface (with landscape crop/scale transform). Each render pass uses a different vertex/uv transform matrix; the fragment shader is the same (oriented texture sampler).
4. **`EncoderSurface`** (internal) тАФ wraps a `MediaCodec` video encoder. Creates the encoder, configures the format, calls `createInputSurface()`, returns the Surface to the EglRouter. Manages the encoder's output buffer drain loop (a daemon coroutine that pulls encoded samples and passes them to the muxer).
5. **`DualMuxer`** (internal) тАФ manages a paired `MediaMuxer` for the two output files. State machine: `INIT тЖТ ADDING_TRACKS тЖТ STARTED тЖТ STOPPING тЖТ RELEASED`. Methods: `addVideoTrack(side, format): Int`, `addAudioTrack(format): Pair<Int, Int>` (returns both muxers' track indices), `start()`, `writeVideo(side, buffer, info)`, `writeAudio(buffer, info)` (broadcast), `stop()` (tolerant of partial failure), `release()`.
6. **`DualMuxerStateMachine`** (internal, pure) тАФ extracted state-machine reducer. Pure functions: `reduce(state, event): State` + `validTransitions: Map<State, Set<Event>>`. JVM-testable.
7. **`AudioFanOut`** (internal) тАФ owns the `AudioRecord` + one AAC `MediaCodec` encoder. Captures PCM, encodes to AAC, broadcasts each encoded sample to all live muxers via `buffer.duplicate()` per-write.
8. **`AudioFanOutPlanner`** (internal, pure) тАФ pure state machine for the broadcast routing decisions: which muxers are live, when to skip a stopped muxer, EOS handling. JVM-testable.
9. **`BitrateTable`** (internal, pure) тАФ maps a `Size` (output resolution) to a bitrate in bps, aligned with `StorageEstimator.bytesPerSecondForResolution` so peak-bytes math stays consistent. Returns `landscape-bps` and `portrait-bps` from a chosen preset. JVM-testable.
10. **`RotationCalculator`** (internal, pure) тАФ pure helper: given `displayRotation` (`Surface.ROTATION_*`) + `VideoSide` (PORTRAIT or LANDSCAPE), returns the integer that gets stored as `MediaFormat.KEY_ROTATION` on the encoder's output. Portrait side mirrors `computeTargetRotation(displayRotation, "Portrait")`; landscape mirrors `computeTargetRotation(displayRotation, "Landscape")`. JVM-testable.
11. **`SegmentPathBuilder`** (internal, pure) тАФ builds the paired output `File`s from a session directory + segment ordinal: `segment_NNNN_P.mp4` + `segment_NNNN_L.mp4`. JVM-testable.

### Lifecycle

1. **Construct** `DualVideoRecorder(config)`. Allocates no native resources; just stores the config. Throws `IllegalArgumentException` from `DualVideoRecorderConfig.init` if config is invalid.
2. **`asCameraEffect()`** returns a `CameraEffect` (with target = `CameraEffect.PREVIEW`) wrapping the internal `DualSurfaceProcessor`. 6.1b adds this to a `UseCaseGroup` and calls `bindToLifecycle`. CameraX delivers the camera frames into the SurfaceProcessor.
3. **`start(outputs, executor, callback)`** opens the muxers, prepares the encoders, fires `DualRecordEvent.Start`. Returns a `DualRecording` handle. The `DualSurfaceProcessor` was already wired to the encoder surfaces when the CameraEffect was bound; `start()` activates the writes.
4. **Frame loop**: CameraX delivers a camera frame via `SurfaceTexture.OnFrameAvailableListener`. EglRouter renders to all three output surfaces. Each encoder's output drain loop pulls encoded samples; muxer writes them. `Status` events are emitted at a throttled cadence (every ~250 ms) with cumulative bytes per side + elapsed ms.
5. **`stop()`** on the `DualRecording`: signals EOS to both video encoders + audio encoder, waits for drain, stops both muxers, releases all resources. Fires `DualRecordEvent.Finalize(portraitFile, landscapeFile, error)`. If both files were written successfully, `error == null`. If one or both failed mid-write, `error` is set and the failed side's `File?` is `null`.
6. **`release()`** on the `DualVideoRecorder`: idempotent. Unbinds anything still bound. After this, the recorder cannot be re-used; create a fresh one.

### Why CameraEffect + EGL fan-out (not finalize-time crop, not full Camera2 rebuild)

Decision rationale captured in the brainstorm:

- **End-user quality**: finalize-time crop produces a Portrait clip with only 56% of source horizontal pixels (e.g., 608├Ч1080 from FHD source). Below social-platform minimums (Instagram Reels / TikTok / Shorts: 1080├Ч1920). Forces UHD source to get usable portrait тЖТ 5├Ч storage cost + thermal + battery harm.
- **EGL fan-out** at the encoder layer produces TWO independent files at their target resolutions. Each is genuinely 9:16 portrait or 16:9 landscape video.
- **CameraX retained** for lifecycle, lens switching, FGS integration, AsyncCloser, recovery patterns. Only the encoder layer is custom. NOT a Camera2 rebuild.
- **Foundation for future features** that need a dual-encoder primitive: live split-screen composite, streaming dual outputs, per-zone effects, pro vertical-platform publishing. Finalize-time crop locks Rova out of all of those.

### Why isolated (Option A) тАФ zero RovaRecordingService diff in 6.1a

- **End-user POV**: 6.1a is invisible. Only matters that existing Portrait/Landscape recordings don't regress. Zero service-file change = zero risk to the user's daily-use feature.
- **No dormant code paths**: feature flags introduce dead branches that reviewers must verify dead. Isolated package has no dead code in `RovaRecordingService`.
- **6.1b is where end-user impact happens**: review-gate focuses on user semantics at 6.1b. 6.1a's review-gate focuses on subsystem correctness тАФ narrower, easier.
- **Reversible**: if a 6.1a flaw is discovered, deleting the package is a one-line rollback. No service-file revert needed.

## 3. Data flow

1. **Construction**: 6.1b code (in 6.1b time; NOT shipped in 6.1a) creates `DualVideoRecorder(config)`. Config carries `cameraInputSize`, `portraitOutputSize`, `landscapeOutputSize`, per-side bitrate, codec, audio settings, lens, rotation, fps. `init` validates.

2. **CameraEffect binding** (6.1b only): `recorder.asCameraEffect()` returns a CameraEffect with target=`CameraEffect.PREVIEW`. The CameraEffect wraps `DualSurfaceProcessor`. 6.1b adds this to a `UseCaseGroup.Builder().addUseCase(preview).addEffect(effect).build()` and calls `cameraProvider.bindToLifecycle(...)`.

3. **CameraX delivers `SurfaceRequest`** to the SurfaceProcessor via `onInputSurface(SurfaceRequest)`. The processor creates an EGL `SurfaceTexture`, provides its Surface to CameraX. CameraX starts pumping frames.

4. **CameraX delivers `SurfaceOutput`** for the PREVIEW target via `onOutputSurface(SurfaceOutput)`. The processor records PreviewView's expected output surface. EglRouter now knows where to render the "preview" copy.

5. **`start(outputs, executor, callback)`** is invoked. Encoders + muxers + audio are prepared:
   - `EncoderSurface(side=PORTRAIT)` and `(side=LANDSCAPE)` each create their `MediaCodec` with the configured size + bitrate, call `createInputSurface()`, register their Surface with EglRouter as additional render targets.
   - `DualMuxer.create(portraitFile, landscapeFile)` opens both muxers (INIT тЖТ ADDING_TRACKS).
   - Each encoder feeds its output-buffer-drain coroutine. First buffer is `INFO_OUTPUT_FORMAT_CHANGED` тАФ the encoder's actual output `MediaFormat` is now known; we call `DualMuxer.addVideoTrack(side, format)` for that muxer. With `MediaFormat.KEY_ROTATION = rotationCalculator(displayRotation, side)`.
   - `AudioFanOut.start()` creates `AudioRecord` (CAMCORDER, 48 kHz mono, PCM16), starts the AAC encoder. Its output drain calls `DualMuxer.addAudioTrack(format)` once, then `DualMuxer.writeAudio(buf, info)` for each subsequent sample (broadcast).
   - When all three tracks are added (both video + audio), `DualMuxer.start()` is called. Both muxers transition to STARTED. The next `writeVideo` / `writeAudio` calls go through.
   - `DualRecordEvent.Start` is delivered to the callback.

6. **Per-frame loop**: `SurfaceTexture.OnFrameAvailableListener` fires on each camera frame. EglRouter does three render passes:
   - To PreviewView's surface (identity transform): displays the camera frame to the user.
   - To encoder-P's input surface (portrait transform: rotate-90┬░ + crop to 9:16 + scale to portraitOutputSize): produces a frame for the portrait encoder.
   - To encoder-L's input surface (landscape transform: identity + scale to landscapeOutputSize): produces a frame for the landscape encoder.
   - Each render pass's `eglSwapBuffers` is the trigger for the encoder to receive the frame.
   - The mirror is applied in the vertex shader when `lensFacing = FRONT` (flips uv.x).

7. **Encoder drain**: each `EncoderSurface` runs a daemon coroutine that pulls encoded samples from MediaCodec's output buffer queue and writes them to the appropriate muxer track via `DualMuxer.writeVideo(side, buffer, info)`. Sample timestamps are MediaCodec-assigned (microseconds since session start).

8. **Audio drain**: `AudioFanOut`'s AAC encoder runs a similar drain coroutine. Each encoded sample is broadcast to both muxers via `DualMuxer.writeAudio(buffer, info)`. The audio encoder + drain run on a separate coroutine from the video drains.

9. **Status events**: throttled to ~250 ms cadence. Captures cumulative bytes written per side + elapsed wall-clock ms. Delivered via callback.

10. **`stop()`** on `DualRecording`:
    - Stop frame ingestion: detach EglRouter's listener from the camera SurfaceTexture (no more renders queued).
    - Signal EOS to both video encoders + audio encoder via `MediaCodec.signalEndOfInputStream()`.
    - Wait for each encoder's drain to finish (until they return `INFO_BUFFER_FLAG_END_OF_STREAM`).
    - `DualMuxer.stop()`: transition to STOPPING, then call `stop()` + `release()` on each `MediaMuxer`. Tolerant: if one throws, log + continue to finalize the other; the failing side's File becomes `null` in the Finalize event.
    - `AudioFanOut.stop()`: stop AudioRecord, stop AAC encoder.
    - Deliver `DualRecordEvent.Finalize(portraitFile?, landscapeFile?, error?)`.

11. **`release()`** on `DualVideoRecorder`: idempotent. Releases EglRouter (GL context, SurfaceTextures), released any encoder/muxer not already released by `stop()`. After release, the recorder is dead.

## 4. Error handling / edge cases

### CameraEffect lifecycle

- CameraX may re-bind the effect when the use-case group changes (e.g., lens switch). The `SurfaceProcessor` receives a fresh `SurfaceRequest`. The EglRouter must handle re-binding: tear down the old SurfaceTexture, create a new one, wire to the same encoder surfaces.
- If `release()` was called before rebind: throw `IllegalStateException` from the SurfaceProcessor's `onInputSurface` (cannot revive a released recorder).

### Encoder configuration failure

- `MediaCodec.configure(format)` can throw if the device doesn't support the requested (codec, profile, resolution, bitrate, fps) combo. Captured during `start()` and surfaced as `DualRecordEvent.Finalize(..., error = ConfigureException)`. No file is created.
- 6.1b consumer should pre-validate using `BitrateTable.isSupported(size, codec)` (a check we don't ship in 6.1a beyond the static table; live `MediaCodecList.findEncoderForFormat` validation is a 6.1b enhancement).

### Muxer failure (one side)

- `MediaMuxer.start()` or `writeSampleData()` can throw on storage error / IO error / corrupt buffer. `DualMuxer` catches per-side errors, transitions only the failed side to a `FAILED` substate, continues writing to the live side. On `stop()`, the failed side's `File` resolves to `null` in `Finalize(...)`; the live side resolves to its `File`.
- The first-write-failure semantics is "best effort to save what's been written" тАФ we accept that a partial-failure session yields one usable file plus a partial/missing one.

### Storage exhaustion

- The session's storage budget is preflighted before recording (6.1b owns this тАФ the storage estimator math will need to account for 2├Ч bitrate at the session-preflight level; expected to be a small change to `estimatePeakBytes`).
- During recording, if a muxer's `writeSampleData` throws `IOException("No space left on device")`, the side fails per the muxer-failure semantics above.

### EGL context loss

- Compose/Android can occasionally destroy the EGL context (process backgrounded + Android low-mem). EglRouter must detect via `EGL14.eglGetError()` and rebuild the context. The encoder surfaces survive (Surface objects are independent of GL context). The render program is recompiled.
- On context-rebuild failure (very rare): emit `Finalize(..., error = EglLostException)`.

### Audio capture failure

- `AudioRecord` can fail if another app holds the mic. Detected at `start()`: `audioRecord.state != AudioRecord.STATE_INITIALIZED`. Emit `Finalize(..., error = AudioInitException)`. No partial recording attempted (we want symmetric files; without audio neither can ship).
- Mid-recording audio drop: not expected on modern Android (mic is locked while AudioRecord is recording). If it happens, the AAC encoder's input goes silent; the muxer continues writing video; the file has video + truncated audio. Acceptable degradation.

### Lens switch mid-recording

- The existing `flipCamera` guard prohibits mid-recording lens switch. `DualVideoRecorder` inherits this тАФ Phase 6's `setMode` and `flipCamera` are 1:1 symmetric and both no-op mid-rec. The DualSurfaceProcessor handles the inter-recording rebind via CameraX's standard rebind flow.

### Mode switch mid-recording

- Same as lens тАФ silent no-op. 6.1b's `setMode` guard preserves this.

### Buffer-pool exhaustion

- If MediaCodec's input buffer pool gets full (frames produced faster than encoder consumes), `eglSwapBuffers` will block. This is bounded by the encoder's actual consumption rate; expected to self-regulate at the configured fps.

### Encoder rotation tagging

- Each encoder's output `MediaFormat` gets `KEY_ROTATION` set at `addVideoTrack` time. The integer comes from `RotationCalculator.tag(displayRotation, side)`:
  - `PORTRAIT` side: same as `computeTargetRotation(displayRotation, "Portrait")` тАФ i.e., display-rotation passthrough
  - `LANDSCAPE` side: same as `computeTargetRotation(displayRotation, "Landscape")` тАФ i.e., display-rotation + quarter-turn
- Players (VLC, system video player, social uploads) rotate on playback per the tag; no pixel rotation is encoded. Matches existing CameraX behavior.

### Front-camera mirror

- Standard CameraX behavior: front camera preview is auto-mirrored; recorded files are NOT mirrored (so playback shows correct orientation). `EglRouter` mirrors the preview render only (vertex shader `uv.x = 1.0 - uv.x` when `lensFacing = FRONT`). The encoder renders are NOT mirrored. Matches CameraX default.

### Long sessions / loop count

- Each segment is a fresh `DualMuxer` instance (created at segment start, released at segment stop). Encoders are re-used across segments (CameraEffect remains bound; encoders are reset between segments). 6.1b's per-segment lifecycle drives this.

### Time-base coherence

- Video samples timestamps come from MediaCodec output buffer info `presentationTimeUs`. Audio sample timestamps come from the AudioRecord's PCM frame index converted to microseconds. Both share a wall-clock time base (session-relative). Both muxers see the same time-coordinated streams.

## 7. 6.1b integration contract (public API surface)

The 6.1b consumer code will call:

```kotlin
// In RovaRecordingService.setupCamera() at 6.1b time (NOT shipped here):
val config = DualVideoRecorderConfig(
    cameraInputSize     = chosenCameraInputSize,        // 6.1b resolution policy
    portraitOutputSize  = Size(1080, 1920),             // example FHD portrait
    landscapeOutputSize = Size(1920, 1080),             // example FHD landscape
    portraitBitrate     = BitrateTable.forSize(Size(1080, 1920), VideoCodec.H264),
    landscapeBitrate    = BitrateTable.forSize(Size(1920, 1080), VideoCodec.H264),
    videoCodec          = VideoCodec.H264,
    audioBitrate        = 128_000,
    audioSampleRate     = 48_000,
    lensFacing          = if (currentCameraSelector == DEFAULT_FRONT_CAMERA) FRONT else BACK,
    displayRotation     = display.rotation,
    fps                 = 30,
)
currentDualRecorder = DualVideoRecorder(config)

val preview = Preview.Builder().setTargetRotation(display.rotation).build()
val ucg = UseCaseGroup.Builder()
    .addUseCase(preview)
    .addEffect(currentDualRecorder!!.asCameraEffect())
    .build()
cameraProvider.bindToLifecycle(this, currentCameraSelector, ucg)

// At segment start (6.1b):
val seq = currentSegmentCount + 1
val portraitFile  = SegmentPathBuilder.build(sessionDir, seq, VideoSide.PORTRAIT)
val landscapeFile = SegmentPathBuilder.build(sessionDir, seq, VideoSide.LANDSCAPE)
currentDualRecording = currentDualRecorder!!.start(
    DualOutput(portraitFile, landscapeFile),
    recordingExecutor,
    ::handleDualVideoEvent
)

// At segment stop (6.1b):
currentDualRecording?.stop()
currentDualRecording = null

// At service teardown (6.1b):
currentDualRecorder?.release()
currentDualRecorder = null
```

The contract is symmetric to CameraX's `VideoCapture` + `Recorder` + `PendingRecording` + `Recording` + `VideoRecordEvent` model тАФ easier future-maintainer mapping.

## 9. Risks + open questions for 6.1a impl

| Risk | Mitigation |
|---|---|
| EGL14 context creation fails on a specific device. | EglRouter exposes `isReady()` after construction; falling back is 6.1b consumer's responsibility (if needed, render to a black surface or fail-fast). |
| `MediaCodec.configure` throws for an unusual (size, bitrate, codec) combo. | Captured into `DualRecordEvent.Finalize.error`. 6.1b consumer presents a UX error. |
| SurfaceProcessor contract specifics on `onOutputSurface`: does target=PREVIEW guarantee we get PreviewView's surface or could we miss the call? | Pre-flight verification in 6.1a impl: write a smoke test (or test program) early in the impl that confirms `onOutputSurface` is reliably invoked for target=PREVIEW. If unreliable, fall back to D2.B (custom SurfaceProvider) тАФ re-spec on discovery. |
| Frame timing skew between video encoders + audio. | Cross-stream skew is bounded by encoder buffering (typically <50 ms). Acceptable. If observed >100 ms in 6.1b on-device smoke, add a sync gate. |
| `MediaMuxer.writeSampleData` throws for a malformed buffer. | DualMuxer per-side failure isolation (D5); the live side continues. |
| `AudioRecord` permission denied at runtime. | `start()` returns `Finalize` with `error = AudioInitException`. 6.1b consumer surfaces a permission warning. |
| EGL frame rendering CPU/GPU cost on lower-end devices. | 6.1b owns the capability probe + UX response. 6.1a's pipeline is configurable enough to support reduced output sizes / bitrates / fps as a fallback. |

### Open questions (resolve during impl or surface to owner)

- **Q1**: When 6.1b consumes the pipeline, which CameraX `Quality` selector does 6.1b's `Preview.Builder` use? The CameraEffect target=PREVIEW captures whatever CameraX delivers, so the Preview's quality determines the camera input size. This is a 6.1b decision (resolution policy).
- **Q2**: For 4:3 native sensor capture (D3 insight), 6.1b needs to drop the default `AspectRatioStrategy.RATIO_16_9` in favor of `RATIO_4_3`. This is a 6.1b CameraX config change. 6.1a's pipeline supports both aspect ratios via the `cameraInputSize` config.
- **Q3**: Should `DualRecording.pause()` / `.resume()` be part of the 6.1a contract (anticipating future seamless-rebind), or added when needed? **Recommendation: defer** тАФ YAGNI for foundation slice.
- **Q4**: ADR placement (ADR-0008?) for the "Dual-recording architecture: CameraEffect + EGL fan-out" decision. Likely owner's call at 6.1a impl time.
