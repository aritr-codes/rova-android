# Phase 6.1c — True Per-Side Rendering for P+L Mode (DualShot)

**Status:** Spec draft, owner-approved sections 1–5 (2026-05-17 brainstorm).
**Builds on:** PR #23 (`feat/dual-recording-consumer`) Phase 6.1b + smoke-fixes #1–#6, ships separately on a new branch.
**Mockup reference:** `mockups/new_uiux/01-record-home.html` (P+L section, lines 695–860).
**Source-of-truth callouts:** ADR-0008 (Phase 6.1a EGL fan-out), smoke-fix #4 (orientation), smoke-fix #5 (consumer-aspect-aware crop — subsumed by 6.1c).

---

## 1. Problem statement

Phase 6.1b shipped P+L recording end-to-end (capture → dual encoders → merge → 2 library cards → playback). Smoke-fixes #4 and #5 corrected orientation metadata and per-side render rotation. But P+L outputs still show aspect-ratio stretch when phone is held in landscape, AND the preview surface is a single shared `PreviewView` — neither side of the output gets its own "correctly framed" preview to verify what's being recorded.

Two root causes:

1. **CameraX center-crops the wide 16:9 sensor to the consumer surface aspect.** When `PreviewView` is portrait, CameraX picks a portrait consumer dim (e.g. 1080×1920) and discards left+right of the sensor view before our shader ever sees the pixels — both encoder outputs are then forced to share a portrait crop, and the landscape encoder ends up showing that portrait crop stretched into a 16:9 frame. ~44% horizontal FOV lost.

2. **One preview surface (`PreviewView`) for both modes.** The user can't visually confirm what each side's recording will look like; the "P+L preview" is just whatever the single PreviewView resolves to. Owner's mockup (`01-record-home.html`) calls for **two stacked preview zones** (portrait on top, landscape below) — each rendering its side's actual framing in real time.

This spec defines Phase 6.1c — the DualShot pattern proper. Capture once from the rear sensor at full FOV, software-fan-out into four render targets (2 previews + 2 encoders), each with its own crop+rotation to produce correctly-framed output.

## 2. Goals + non-goals

### Goals

- **G1.** Both P+L output files render correctly framed for their intended aspect — no stretch. Portrait file = 9:16 center-crop of the 16:9 source. Landscape file = full 16:9 source.
- **G2.** Record-home P+L mode shows two stacked preview zones matching `mockups/new_uiux/01-record-home.html`. Each zone shows live preview of its side's framing during BOTH idle and active-recording states.
- **G3.** Preview aspect inside each zone is letterboxed to the true output aspect — user sees exactly what gets recorded.
- **G4.** Pinned-orientation behavior preserved — recording orientation is captured at session start and frozen for the session (no mid-recording rotation handling).
- **G5.** Single-mode Portrait and Landscape pipelines untouched. No regression for non-P+L users.
- **G6.** Carry-over invariants from PR #23 unaffected: warning system, recovery system, manifest schema, history flatMap, notification clip count, flip-camera-gated-in-P+L, AudioFanOut/DualMuxer/EncoderSurface bytecode-stable.

### Non-goals

- **NG1.** Front camera P+L. Owner scoped out at smoke-fix #6 — front-cam-with-P+L combination is gated. Phase 6.1c does not re-enable.
- **NG2.** 4K landscape consumer surface. Phase 6.1c pins to 1920×1080. Higher resolutions are a future device-tier expansion.
- **NG3.** Mid-session orientation change (rotate phone while recording). Encoder dims are fixed at session start; same as Phase 6.1b.
- **NG4.** Concurrent dual physical camera (back-wide + back-ultrawide simultaneously). Not viable on the target device class (Samsung A17 and entry-level Android). The DualShot pattern from one full-FOV sensor frame IS the canonical software approach for these devices.
- **NG5.** Aspect-ratio choices other than 9:16 and 16:9 (e.g., 1:1 square, 4:3, ultrawide 21:9). Future modes if requested.
- **NG6.** `AudioFanOut.stop()` invalid `signalEndOfInputStream` call cleanup (noise reduction follow-up, not 6.1c scope).
- **NG7.** Filename millisecond mismatch (`_482` vs `_783`), ADR-0008 "activated" stamp (post-merge doc-hygiene), and other parked follow-ups.

### 4.3 TextureView lifecycle (each zone)

```kotlin
AndroidView(
    factory = { ctx ->
        TextureView(ctx).apply {
            surfaceTextureListener = object : TextureView.SurfaceTextureListener {
                override fun onSurfaceTextureAvailable(st: SurfaceTexture, w: Int, h: Int) {
                    val surface = Surface(st)
                    registerPreviewSurface(side, surface, w, h)
                }
                override fun onSurfaceTextureSizeChanged(st: SurfaceTexture, w: Int, h: Int) {
                    // Re-register with new dims; service-side maps key is `side` so this
                    // replaces the prior entry. EglRouter recomputes viewport.
                    val surface = Surface(st)
                    registerPreviewSurface(side, surface, w, h)
                }
                override fun onSurfaceTextureDestroyed(st: SurfaceTexture): Boolean {
                    unregisterPreviewSurface(side)
                    return true  // let SurfaceTexture be released by the framework
                }
                override fun onSurfaceTextureUpdated(st: SurfaceTexture) = Unit
            }
        }
    },
    modifier = Modifier.fillMaxSize()
)
```

The composable is internal-only; doesn't expose the SurfaceTexture directly to its caller — only the side-tagged register/unregister callbacks.

## 5. Service + render layer

### 5.1 `setupDualCamera()` resolution change

```kotlin
// Phase 6.1c — force landscape 1920×1080 consumer for full sensor FOV.
val landscapeRes = android.util.Size(1920, 1080)
val resolutionSelector = ResolutionSelector.Builder()
    .setResolutionStrategy(
        ResolutionStrategy(
            landscapeRes,
            ResolutionStrategy.FALLBACK_RULE_CLOSEST_HIGHER_THEN_LOWER,
        )
    )
    .setAspectRatioStrategy(AspectRatioStrategy.RATIO_16_9_FALLBACK_AUTO_STRATEGY)
    .build()

    .setResolutionSelector(resolutionSelector)
    .setTargetRotation(android.view.Surface.ROTATION_0)
    .build()
```

`setTargetRotation(ROTATION_0)` keeps the camera producing sensor-native landscape — no CameraX auto-rotation. All rotation correction happens in our shader (Section 5.4).

PreviewView still bound via `setSurfaceProvider(surfaceProvider)`. Its Surface gets registered as a render target (legacy path) but EglRouter SKIPS drawing to it in P+L mode (the new `DualPreviewZone` TextureViews cover it visually). Avoids touching the CameraX bind path during rebind safety.

### 5.3 `EglRouter` — `TargetKind` enum, `removeTarget`, per-target viewport

```kotlin
internal enum class TargetKind { ENCODER, PREVIEW }

private data class RenderTarget(
    val side: VideoSide?,          // PORTRAIT, LANDSCAPE, OR null = legacy CameraEffect Preview output
    val kind: TargetKind,
    val surface: Surface,
    val eglSurface: EGLSurface,
    val width: Int,
    val height: Int,
    val cropMatrix: FloatArray,
    val mirrorMatrix: FloatArray,
    var viewportX: Int,
    var viewportY: Int,
    var viewportW: Int,
    var viewportH: Int,
)

fun addTarget(
    side: VideoSide?,
    kind: TargetKind,
    surface: Surface,
    width: Int,
    height: Int,
)

fun removeTarget(side: VideoSide?, kind: TargetKind)
```

`side` stays nullable for the legacy CameraEffect Preview output (from `DualSurfaceProcessor.onOutputSurface`) — registered as `addTarget(side = null, kind = PREVIEW, ...)`. `renderFrame` skips targets matching `(side == null && kind == PREVIEW)` — no draw, no swap, the legacy path is fully inert in 6.1c. The PreviewView is covered visually by the new `DualPreviewZone`; the unused legacy surface costs only one onOutputSurface call at bind time. (Future cleanup: drop the legacy target entirely once the rebind path is confirmed; out of scope for 6.1c.)

`addTarget` body:
- Create EGL window surface for the input Surface.
- If `side != null`: build cropMatrix via `AspectFitMath.buildCropMatrix(displayRotation, side, cropMatrix)`; compute viewport via `AspectFitMath.computeFitViewport(width, height, contentAspect(side))` where `contentAspect(PORTRAIT) = 9f/16f` and `contentAspect(LANDSCAPE) = 16f/9f`.
- If `side == null` (legacy CameraEffect Preview output): cropMatrix stays identity, viewport stays `(0, 0, width, height)`. Values are unused — renderFrame skips this target entirely.
- Build mirrorMatrix = identity (front cam path gated in P+L; mirror is dead code in 6.1c).
- Append RenderTarget.

`removeTarget` body:
- Find target by (side, kind), destroy its EGLSurface, drop from `targets` list.
- Idempotent — missing target is a no-op.

`renderFrame` body (rewritten to use the new per-target viewport):

```kotlin
for (target in targets) {
    // Phase 6.1c — legacy CameraEffect Preview output (side=null+kind=PREVIEW)
    // is inert: covered by DualPreviewZone visually, no draw needed.
    if (target.side == null && target.kind == TargetKind.PREVIEW) continue

    EGL14.eglMakeCurrent(eglDisplay, target.eglSurface, target.eglSurface, eglContext)
    // 1. Clear full surface with black — paints letterbox/pillar bars.
    GLES20.glViewport(0, 0, target.width, target.height)
    GLES20.glClearColor(0f, 0f, 0f, 1f)
    GLES20.glClear(GLES20.GL_COLOR_BUFFER_BIT)
    // 2. Restrict viewport to aspect-fit region and draw the OES sample.
    GLES20.glViewport(target.viewportX, target.viewportY, target.viewportW, target.viewportH)
    GLES20.glUseProgram(program)
    bindOesTexture()
    Matrix.multiplyMM(tmpMatrix, 0, texMatrix, 0, target.mirrorMatrix, 0)
    Matrix.multiplyMM(finalMatrix, 0, target.cropMatrix, 0, tmpMatrix, 0)
    GLES20.glUniformMatrix4fv(uTexMatrixLoc, 1, false, finalMatrix, 0)
    drawQuad()
    EGL14.eglSwapBuffers(eglDisplay, target.eglSurface)
}
```

The per-frame `computeCropMatrix` introduced in smoke-fix #5 is REMOVED — cropMatrix is built once at `addTarget` time using the session-pinned `displayRotation`, then reused every frame. Cheaper, simpler, deterministic.

### 5.4 `AspectFitMath` — pure-JVM math seam

```kotlin
internal object AspectFitMath {
    /**
     * Aspect-fit viewport inside a surface of [surfaceW] × [surfaceH] for
     * content of aspect [contentAspect] (= width/height). Returns
     * [x, y, w, h] for glViewport. Surface dims must be positive;
     * contentAspect must be > 0.
     */
    fun computeFitViewport(surfaceW: Int, surfaceH: Int, contentAspect: Float): IntArray

    /**
     * cropMatrix = sideAspectCrop[side] × displayRotationCorrection(displayRotation).
     * Both factors built as 4×4 column-major matrices, composed via Matrix.multiplyMM.
     */
    fun buildCropMatrix(displayRotation: Int, side: VideoSide, out: FloatArray)

    /**
     * Pivot-rotate UV around (0.5, 0.5) by N° where N depends on
     * displayRotation. See KDoc for the sign table:
     *   0 → +90°    (phone portrait, sensor landscape: rotate to portrait-up)
     *   1 → 0°      (phone landscape: identity)
     *   2 → +270°   (phone upside-down portrait)
     *   3 → +180°   (phone landscape other-way)
     * Signs verified on-device per smoke-fix tradition; flip in this seam if needed.
     */
    fun buildDisplayRotationCorrection(displayRotation: Int, out: FloatArray)

    /**
     * sideAspectCrop:
     *   PORTRAIT  → pivot-scale UV around (0.5, 0.5) by (9/16, 1, 1) — center-crop horizontal column.
     *   LANDSCAPE → identity.
     */
    fun buildSideAspectCrop(side: VideoSide, out: FloatArray)
}
```

All four functions are pure (no Android dependencies) — JVM-unit-testable.

## 7. Migration + risk

### 7.1 Branch / PR strategy

- **PR #23 (Phase 6.1b) ships its current state first** (tip `86a5b46`, smoke-fix #6) — verified P+L beta with documented stretch limitation. Owner-gated.
- **Phase 6.1c starts on `feat/phase-6.1c-dualshot-render`** — new branch off `master` AFTER 6.1b merges. Own spec (this doc), own plan, own PR.
- **No data migration** — manifest schema, segment files, recovery path, exporter tier dispatch all unchanged.

### 7.2 Cleanup / deletion in 6.1c

- `SplitPreviewOverlay` composable — DELETE (labels live inside `DualPreviewZone`'s `cam-zone-tag`s now).
- `EglRouter.computeCropMatrix(target)` per-frame call (smoke-fix #5) — REMOVE (cropMatrix is built once at addTarget using pinned displayRotation; no per-frame recompute).
- Legacy `EglRouter.addTarget(side=null, ...)` for CameraEffect Preview output — either keep as no-draw target (Section 5.3 chosen path) OR drop entirely once rebind path is verified (follow-up).

### 7.3 Risk register

| Risk | Likelihood | Mitigation |
|---|---|---|
| Sign of displayRotation correction wrong on first device run | Medium | Single-constant flip in `AspectFitMath.buildDisplayRotationCorrection`; smoke-fix tradition (#4, #5 both had sign-verify clauses); on-device gate Section 6.3 step 8 |
| TextureView SurfaceTexture race vs camera flip | Medium | `pendingPreviewSurfaces` map replay in `setupDualCamera` (Section 5.2); `attach/detach` idempotent in `EglRouter.removeTarget` |
| ResolutionSelector picks non-1920×1080 closest match on some devices | Low | `FALLBACK_RULE_CLOSEST_HIGHER_THEN_LOWER` + `consumerWidth/Height` runtime values flow through to cropMatrix correctly |
| PreviewView legacy target costs an extra eglSwapBuffers/frame | Very low | Acceptable for now; swap to dummy SurfaceProvider in P+L if profiling shows issue |
| TextureView destroyed without detach (leak) | Low | `onSurfaceTextureDestroyed` returns true AND calls `unregisterPreviewSurface` → `detachPreviewInput` → `removeTarget` |
| Compose AndroidView recomposition recreates TextureView mid-session | Low | `factory` lambda runs once; `update` lambda not used. Stable across recompositions |
| Smoke-fix #5's per-frame `computeCropMatrix` removal breaks something | Low | The per-frame logic was a transition step; pinning at session start matches the encoder dim pinning contract |
