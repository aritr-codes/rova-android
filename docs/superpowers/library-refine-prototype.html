Annotations — Default state

M1Filter affordance. Active additive toggles (Favorites, DualShot) gain a leading ✓; All stays check-less and is separated by a hairline gap. The asymmetry breaks the "segmented tab bar" read — it now says All = reset, the other two = stackable. Two checks max, only on chips you activated → no noise.
M3aTile density. Orientation glyph is removed only on DualShot tiles (P+L is both framings — the glyph is ambiguous there, and these are the exact tiles that stacked 3 TopEnd pills). Single-mode tiles keep the glyph: the tile is locked 16:9, so the thumbnail never reveals source orientation — the glyph is the only orientation signal and earns its place for scanning.
USAGEUsage-line placement. Kept (footprint/prune value is real) but demoted one weight step so it reads as a quiet caption under the title, not a content row competing with the hero. Day headers already carry per-section size, so the global line is a summary, not a peer.
ACTIONSAction hierarchy. Top bar reordered to expected frequency: Search → Sort → View promoted; low-use Vault demoted to the trailing slot and dimmed. Labels shown here only to expose the order — the app stays icon-only.
HEROHero refinement. "LATEST RECORDING" eyebrow (3 lines of identity) compressed to a single small "Latest" corner tag + a slightly taller frame. Frees vertical space, lets the title lead, and widens the hero/tile distinction once M3a lightens the tiles.

Annotations — Item sheet

KEEPThree-zone grouping (Primary / Secondary / Danger) is retained — it already works. Delete is quarantined behind a divider.
SELECTSelect stays in the sheet by design, not inertia. It is categorically a mode-switch, not an item action, and its correct home is a top-bar "Select" — but that's blocked by the selection-architecture constraint (idle "active-but-empty" can't be represented). Removing it now strands discoverability (only other entry is long-press). Verdict: C — reject moving it until that arch work happens.
PLAYPlay emphasis (optional). Refined gives Play a slightly heavier row so the one-tap dominant action leads by weight, not just position. Hierarchy only, no restyle. B (later) — hold if it risks the clean grouping.

Evaluation — does each change earn its place?

QuestionPrototype showsVerdict
Does M1 improve filter clarity?Yes. The ✓-on-toggles + gapped All visibly breaks the tab-bar read; "two are on" becomes self-evident. Minimal added marks.A · ship
Does M3a improve tile readability?Yes, surgically. DualShot tiles drop from 2 pills to 1 and lose the ambiguous glyph; single-mode tiles keep orientation (the only signal of it). Cleaner where redundant, informative where not.A · ship
Should the usage line move?Not move — demote. Kept under the title as a quiet caption; day headers carry the per-section detail. Removing it loses prune value; promoting it competes with the hero.A · demote
Should action hierarchy change?Yes. Search/Sort/View promoted, Vault demoted + dimmed to match real frequency. Low cost, clear win.B · later
Hero — further refinement?Light touch only: trade the 3-line eyebrow for a small "Latest" tag + taller frame. Already distinct; M3a widens the gap for free. No structural change.B · optional

Smallest set that maximizes quality: ship M1 + M2 + M3a (all A). Usage-line demote rides along with M1 (same top-section edit, near-zero cost). Top-bar reorder, hero tag, Play emphasis, and Delete isolation are B — worth doing, but should not gate the theme engine. Item-sheet Select move is C — blocked by arch, reject for now.
